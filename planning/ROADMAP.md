# ROADMAP — Sarhad

## Product goal
Ship a polished Android-first portrait strategy RPG loop centered on live travel, encounter decisions, tactical combat, and settlement recovery/progression.

## Success criteria
- New players understand travel -> encounter -> settlement loop in first session.
- Battle/Run/Bribe is consistently used as a meaningful decision point.
- Session pacing supports both 3–8 minute and 10–20 minute play.
- Stable performance on target mid-tier Android phones.

## Phase 1 — Core loop alignment (current docs-first pivot)
Focus:
- Lock docs and terminology to Sarhad concept.
- Keep implementation scope centered on live travel + encounter decisions + settlements.
- Ensure resource model (gold/supplies/troops/time) is explicit everywhere.

Exit criteria:
- Planning docs are internally consistent with Sarhad concept.
- No conflicting old concept language in core planning pages.

## Phase 2 — Playable vertical slice
Focus:
- Portrait overworld with joystick travel and moving parties.
- Encounter decision screen (Battle / Run / Bribe).
- Short real-time battle with core commands.
- Basic settlements: Rest, Recruit, Supplies, Missions.

Exit criteria:
- End-to-end loop playable without manual intervention.
- Battle duration reliably within 30s–3m target.
- Settlement actions close loop after losses/wins.

## Phase 3 — Progression depth
Focus:
- Mission variety (hunt, escort, delivery, rescue, pursuit).
- Better differentiation between towns and villages.
- Economy tuning for bribe/recruit/recovery tradeoffs.
- Stronger warband growth arc across longer sessions.

Exit criteria:
- Multiple viable progression strategies.
- Resource pressure remains meaningful (no dominant exploit loop).
- Session types (short/medium/long) all feel rewarding.

## Phase 4 — Polish and release prep
Focus:
- UI/UX polish and readability pass.
- Onboarding/tutorialization for encounter choices and resource pressure.
- Device compatibility + performance optimization.
- Store readiness and release checklist.

Exit criteria:
- Core loop clarity targets met.
- Performance acceptable on target device tier.
- No blocker bugs in travel, encounter flow, battle resolution, or settlement actions.

## Ongoing cut policy (if schedule risk rises)
Cut in this order:
1. Mission variants beyond core set.
2. Non-critical visual flair.
3. Edge-case encounter variants.
4. Secondary progression layers.
5. Any feature that weakens readability/performance.

## Milestone review questions
- Is "one more stop, one more mission" still strong?
- Are battle/run/bribe choices still meaningful?
- Is survival pressure still felt (troops, supplies, gold, time)?
- Is portrait one-handed usability still intact?
- Did added content reduce clarity or frame stability?
